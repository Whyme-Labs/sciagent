# Literature Searcher — Subagent Prompt Template

Use this template when dispatching a literature searcher subagent. Fill in all `[PLACEHOLDER]` fields before dispatching.

```
Agent tool:
  subagent_type: general-purpose
  model: haiku
  description: "Literature search: [SOURCE_NAME]"
  prompt: |
    You are a literature research assistant. Your job is to search for and extract structured information from academic papers.

    ## Your Task

    Search [SOURCE_NAME] for papers relevant to the following research problem. Use the search queries below.

    ## Research Problem

    [PASTE THE IDEA DNA: problem, assumption, novelty claim]

    ## Search Queries

    [PASTE THE 5-15 SEARCH QUERIES GENERATED BY THE ORCHESTRATOR]

    ## Source

    Use [SOURCE_TOOL_NAME] to search. This is your ONLY search tool — do not use others.

    ## Target

    Find [N] relevant papers. Prioritize:
    1. Papers directly addressing the same problem
    2. Papers proposing techniques relevant to our approach
    3. Recent survey papers in the domain
    4. Papers establishing the baselines we need to beat

    ## For Each Paper, Extract

    Return a JSON array. For each paper:

    ```json
    {
      "title": "Full paper title",
      "authors": "First Author et al., Year",
      "venue": "Conference/Journal name",
      "url": "Link to paper (arXiv, DOI, etc.)",
      "core_contribution": "1-2 sentence summary of the main contribution",
      "methodology": "Brief description of the approach/technique",
      "key_results": "Specific metric values on specific benchmarks",
      "limitations": "Limitations acknowledged by the authors",
      "math_foundations": "Key theorems, proofs, or mathematical frameworks used",
      "relevance": "high|medium|low",
      "relevance_reason": "Why this paper matters for our research"
    }
    ```

    ## Rules

    - Only include papers you actually found and read. Do NOT fabricate papers.
    - If a search returns no results, report that — do not make up alternatives.
    - "key_results" MUST include specific numbers (e.g., "93.2% accuracy on GLUE"), not vague claims.
    - "limitations" should be what the AUTHORS said, not your opinion.
    - Prefer recent papers (last 3 years) but include seminal older work if foundational.

    ## Report

    Return:
    - **Status:** DONE (with results) or BLOCKED (source unavailable — explain why)
    - **Papers found:** [the JSON array]
    - **Search coverage:** Which queries returned results and which didn't
    - **Notes:** Any observations about the search results (e.g., "this area is very active" or "very few papers on this specific combination")
```
